01 Overview

Lightweight correction for action-chunked VLA policies.

Action chunks reduce policy-call frequency by executing several predicted actions before the next VLA query. This improves smoothness and efficiency, but it creates an open-loop blind spot: fresh observations arrive while the robot continues following queued actions.

VLA-Corrector adds an external latent dynamics corrector trained on frozen VLA visual features. It detects persistent mismatch between expected and observed visual evolution, truncates stale actions, and guides the next recovery query with Online Gradient Guidance.

~40M Residual MLP corrector, reported as approximately 38--42M parameters.
3 Evaluated VLA backbones in the paper: PI0.5, SmolVLA, and X-VLA.
83.7% Reported truncations occurring in manually labeled critical phases.
02 The Problem

Open-loop VLA execution leaves blind spots.

Open-loop VLA

Long chunks can become stale.

  • Fresh observations are available but not used until the horizon ends.
  • Small pose drift, slippage, or disturbance can compound during the blind spot.
  • A later ordinary replan may start from a harder out-of-distribution state.
VLA + Corrector

Correction is invoked only when needed.

  • LVM monitors whether local visual dynamics remain consistent.
  • Persistent mismatch truncates the remaining queued actions.
  • OGG guides the next recovery query without retraining the full VLA.
03 The Idea

Correct the action, not the whole model.

VLA-Corrector decouples action generation from execution monitoring. The base policy continues to generate action chunks, while a separate corrector estimates local latent dynamics and decides whether the current chunk should still be trusted.

04 Method

Four modules turn fixed horizons into event-triggered horizons.

The method keeps standard chunked execution in stable phases and activates correction only after the monitor detects persistent drift.

01

External Latent Dynamics Corrector

Predicts short-horizon latent residuals from frozen VLA visual features and executed actions.

02

Latent-space Vision Monitor

Compares expected and observed visual evolution while the action chunk is being executed.

03

Event-triggered Truncation

Uses robust online thresholds, hysteresis, and persistence checks before discarding stale actions.

04

OGG Recovery Replan

Applies Online Gradient Guidance only to the single policy query immediately after an interrupt.

06 Usage

Install, train the corrector, and evaluate with your own resources.

Datasets, pretrained weights, fine-tuned checkpoints, trained corrector checkpoints, demo data, and logs are not included in this repository.

Installation
conda env create -f environment.yml
conda activate lerobot
python -m pip install -e . --no-build-isolation
Corrector training
torchrun --nproc_per_node=1 -m siglip_dynamics.train \
  --model-type mlp \
  --h-window 1 \
  --k-step-list 10 \
  --dataset-path <EXTRACTED_CACHE_DIR> \
  --batch-size 512 \
  --epochs 30 \
  --train-loss-type cosine \
  --checkpoint-dir <CORRECTOR_CHECKPOINT>
Modified evaluation
python -m lerobot.scripts.lerobot_eval_modified_detection \
  --policy.path=<POLICY_CHECKPOINT> \
  --policy.device=cuda \
  --env.type=metaworld \
  --env.task=<TASK_SPLIT> \
  --eval.batch_size=1 \
  --eval.n_episodes=20 \
  --safety_model_path=<CORRECTOR_CHECKPOINT> \
  --output_dir=<OUTPUT_DIR>
