VLA-Corrector

VLA-Corrector: Lightweight Detect-and-Correct Inference for Adaptive Action Horizon

A lightweight ~40M external latent dynamics corrector that monitors open-loop VLA execution, interrupts stale action chunks, and guides recovery replanning without retraining the frozen VLA backbone.

Yi Pan, Miao Pan, Qi Lu, Jiaming Huang, Man Zhang, Wenqi Zhang
Zhejiang University

Code README Paper: Coming soon arXiv: Coming soon BibTeX

Abstract

Detect drift, truncate stale chunks, and replan toward recovery.

Action-chunked VLA policies reduce policy-call frequency by executing several predicted actions before querying the model again. This creates an open-loop blind spot: fresh observations arrive, but the policy keeps following queued actions until the fixed horizon ends. VLA-Corrector adds a lightweight detect-and-correct pathway that monitors latent visual dynamics, detects persistent drift, truncates stale actions, and applies Online Gradient Guidance to the next recovery replan.

Motivation

Mitigating blind spots in open-loop VLA control

Why action chunks drift

Larger action horizons amortize expensive VLA inference, but the robot may continue executing stale actions after slippage, pose drift, collision, or online disturbance. The paper identifies this as an open-loop blind spot in action-chunked VLA policies.

Why not query every step?

Strict closed-loop execution with horizon 1 improves responsiveness, but it requires a full VLA inference at every control step. VLA-Corrector instead makes the action horizon adaptive: keep long chunks when reliable, and intervene when drift persists.

Corrector Training

Train an external residual MLP on frozen VLA visual latents to predict short-horizon latent residuals.

LVM Detection

Compare expected and observed latent visual evolution from fresh camera observations.

Event Trigger

Use robust thresholds, hysteresis, and persistence checking before truncating stale actions.

OGG Recovery

Guide only the recovery query after an interrupt toward a corrective latent direction.

Corrected execution

From fixed open-loop chunks to event-triggered correction

VLA-Corrector does not claim to replace the base policy with a full new closed-loop controller. Instead, it adds a lightweight correction pathway: standard chunked execution remains active during stable phases, while LVM-triggered truncation and OGG-guided replanning intervene at recovery moments.

Ablation and analysis

Where the gains come from

Truncation matters

On the reported MetaWorld ablation, truncation alone improves average success from 48.70% to 60.35%, showing the value of stopping stale chunks.

Guided recovery helps

Adding OGG raises the reported average to 64.35%. The paper also notes that OGG adds event-triggered wall-clock overhead because it uses online gradients during recovery queries.

Usage

Install, prepare your data and checkpoints, then run evaluation

conda env create -f environment.yml
conda activate lerobot
python -m pip install -e . --no-build-isolation
python -m lerobot.scripts.lerobot_eval_modified_detection \
  --policy.path=<POLICY_CHECKPOINT> \
  --policy.device=cuda \
  --env.type=metaworld \
  --env.task=<TASK_SPLIT> \
  --eval.batch_size=1 \
  --eval.n_episodes=20 \
  --safety_model_path=<CORRECTOR_CHECKPOINT> \
  --safety_k=10 \
  --guidance_eta=1 \
  --output_dir=<OUTPUT_DIR>
